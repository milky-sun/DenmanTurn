为了让你更安心，我为你拆解这两个警告的具体原因：

### 1. `警告: [options] 未与 -source 8 一起设置引导类路径`

* **原因**：你的电脑上安装的 Java 编译器（javac）版本比较新（比如 Java 11, 17, 21 等），但我们在脚本里强制指定了 `-source 1.8`（即 Java 8 语法），以便兼容 Android 的旧运行时。
* **翻译**：编译器在发牢骚：“嘿，你让我用 Java 8 的标准去检查代码，但你没给我 Java 8 的标准库文件（rt.jar），我只能暂时用现在的库凑合一下。”
* **为什么没事**：因为我们通过 `-cp "$ANDROID_JAR"` 强行引入了 Android 的核心库。在 Android 开发中，`android.jar` 才是真正的“老大”，只要它没报错，Java 编译器的这个警告就可以无视。

### 2. `注: ...使用或覆盖了已过时的 API`

* **原因**：我们的代码为了兼容旧手机，必须使用一些“过时”的方法。
* **具体实例**：
* 我们在 `RotationService.java` 里写了 `WindowManager.LayoutParams.TYPE_PHONE`。这个东西在 Android 8.0 就被标记为“过时（Deprecated）”了，Google 推荐用 `TYPE_APPLICATION_OVERLAY`。
* **但是**，为了让你的 App 能在 Android 7.0 上运行，我们**必须**写这个过时的代码。


* **安全性**：我们在代码里已经加了 `if (Build.VERSION.SDK_INT >= ...)` 判断。
* 如果是新手机，代码会自动走新 API。
* 如果是旧手机，代码会走旧 API。
* 编译器比较笨，它只要看到旧 API 就会喊一句“过时了”，但它不知道我们已经做了完美的兼容性处理。
